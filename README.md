# Лаба 3 — P2P чат (Socket API)

Консольный одноранговый чат без WebSocket.

- Обнаружение узлов: UDP broadcast.
- Обмен сообщениями: TCP.
- Каждый узел одновременно слушает входящие подключения и может подключаться к другим узлам.
- IP и порты задаются пользователем (CLI или интерактивный ввод).
- При запуске проверяется доступность UDP/TCP порта (ошибка bind выводится явно).
- UI не блокируется сетью: все сетевые операции идут в фоновых потоках.

## Запуск

```bash
python3 p2p_chat.py --name Alice --bind-ip 127.0.0.1 --udp-port 50000 --tcp-port 50001
```

Можно без параметров — программа спросит значения интерактивно:

```bash
python3 p2p_chat.py
```

## Команды в чате

- `/help` — справка
- `/peers` — активные TCP-соединения
- `/history` — история событий с момента запуска
- `/quit` — выход
- `<текст>` — отправить сообщение всем узлам

## Как тестировать на одном ПК

Запусти несколько экземпляров с разными loopback-IP и TCP-портами:

Терминал 1:
```bash
python3 p2p_chat.py --name Alice --bind-ip 127.0.0.1 --udp-port 50000 --tcp-port 50001
```

Терминал 2:
```bash
python3 p2p_chat.py --name Bob --bind-ip 127.0.0.2 --udp-port 50000 --tcp-port 50002
```

Терминал 3:
```bash
python3 p2p_chat.py --name Carol --bind-ip 127.0.0.3 --udp-port 50000 --tcp-port 50003
```

Все узлы должны обнаружить друг друга по UDP и связаться по TCP.

## Формат TCP-сообщений

Используется собственный фрейминг (рекомендация из задания):

- 1 байт: тип сообщения
- 4 байта: длина payload (big-endian)
- N байт: JSON payload

Типы:
- `1` — чат-сообщение
- `2` — hello (идентификация узла)

## Что пишется в истории

С отметкой времени:
- обнаружение нового узла;
- установка/закрытие TCP-соединений;
- входящие сообщения (имя и IP отправителя);
- собственные отправленные сообщения;
- уведомления об отключении узлов (по UDP leave).
